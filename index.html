当时我只是不想学习

你的车载上我总会鸣笛，你怨我太重，其实是你的车太小。 
记得你带上我，骑着小小的自行车，说就以这点开始游湛江。我说全靠你了，结果你带我绕圈圈，说不知道路，结果我一说那把你卖了养我，你却说不值钱。

你很会玩，叫我在海滩上挖坑，说会汇成一口井，结果我们都没动手。

你在沙滩上骑着小小的车，结果不得不下来。好像跟你在一起的时光过得总是特别特别的快。也许是你的话太多，嘀嘀咕咕地讲啊讲，所以时光过得很快。

你说的话总是很甜，我不爱听你贫，所以我一直都是直接忽略你。可后来回头看看，如今的你真实多了，没有那么贫了。

你很恶心，当着我姐的面用我新买的衣服擦嘴，我一生气就扭你打你，后来我有开始考虑你疼不疼了。所以决定以后不打你了。

今天我笑着从你面前跑走，只是不想让你看到我的脆弱和泪水。后来你也一直没来找我，我不知道你是不是完全忘了我在难过。所以我有些许失落。

我记得你总是爱拍着我的头，有点吃的就给我，说吃多点，长高点，就不用仰头看你了。一开始我不喜欢这样的话，后来慢慢地觉得这些话也让我温暖。

姐姐说你会不会很照顾我，我只知道有啥事就跑去找你。只是如今也少了，只是怕打扰到你。

你曾经说什么叫做浪漫，我说不知道，其实当时你用小小的自行车艰难地载着我扫荡街头时，我就觉得很浪漫，只是我没告诉你。

感觉和你在一起的时光很快乐，一起从海关走到国贸，一起在路边坐着等了两个多小时的车，一起骑车游荡，一起吃饭时你的神经，一起漫步时的惬意。虽然我常嫌你太矮，可是姐姐说我和你的高度差很perfect的时候我很高兴。

感觉一切都很美妙，只可惜，找不到心跳加快的感觉，见到你脸红只有一次……希望我们只做一辈子的好朋友……

一图 App 体验报告

前述：看到 Airing 的朋友圈后，第一眼就喜欢上了这个超萌的 UI 设计，所以第一时间就跑去 App Store 下载并体验了。因为不了解 JS 那一块的开发，所以所述建议都是基于普通用户层面的，不作技术实现细节上的探讨。若以下所述有不适当的地方，还望海涵。最终的目的也是希望一图能越做越好。

测试环境：

* 硬件：iPhone 5s (A1530)
* 软件：iOS 10.3.1（14E304）
* 语言与地区：简体中文 中国
* 网络环境：基于广州大学校园网的 Wi-Fi

登陆页

发现的问题：
1、无法单击1处让键盘收起
2、无法像3所示上下滑动来显示密码输入框
3、输完帐号之后，只能单击2处的 return 键来收起键盘，然后再点击密码输入框输入密码

小建议：
1、在点击“帐号输入框”时，整个界面能根据不同机型自动上滑，上滑到键盘不至于遮挡住帐号和密码输入框；
2、在1处的添加动作，实现单击时收起键盘；
3、或者当帐号输入框为第一响应者时，可以把2处的 return 改为 next，并添加事件，当点击 next 时自动把第一响应者切换到密码输入框。

登陆页与主页的切换

其实这个不知算不算问题，就是在主页时可以通过左侧滑来实现退出登陆的操作，有点匪夷所思。退出登陆的操作一般会放在“我的”功能模块里。

一级二级菜单切换

切换过程有些控件（箭头所指）残留在过渡界面上

搜索

单击搜索框进入搜索界面时，无法自动弹出键盘让用户输入。这个也不知道算不算问题，因为单击进入搜索页面时，还有“热门搜索”和“搜索历史”的版块需要展示，如果一进入搜索界面就弹出键盘，就会遮挡了“搜索历史”，造成功能性的冲突。输入关键字搜索时，建议添加关键字自动匹配功能，方便用户模糊查询。

结语：V1.0能做到这个程度，还是很敬佩你们这群大咖，向大咖致敬，也希望你们能继续完善一图，给广大用户带来一个真正意义上的移动图书馆。

自我介绍（小熊篇）

各位面试官，你们好，我叫钟春伶，是一名2013届华南师范大学的应届毕业生，大学专业是地理科学。我将从以下几点进行简单的自我介绍。

作为师范本科生，为了更好地为教师职业做准备，我认真完成相关学业，期间获得了校级二等奖学金，四年专业平均绩点排名5%，并随学院赴内蒙古、江西庐山和湛江等地进行了3次户外专业实习，据此积累了较扎实的专业基础，并掌握了一定的户外地理学习技能。

除了专业知识的学习，我也积极参与到科研当中，在科研过程中不断成长，不断培养自己的创新意识和挑战意识。

此外，大学期间为了锻炼自己的组织能力和领导能力，我还担任了年级学习委员，三年连续担任导师制小组组长，所在小组连获院级优秀学习小组。

常言道，医生救人性命，教师救人灵魂。我深知，只有一个本身渊博的、有趣的、充满能量的和坚定的灵魂才能更好地培养出有火、有梦、有心的无数灵魂。
